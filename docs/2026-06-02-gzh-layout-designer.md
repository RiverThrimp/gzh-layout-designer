# 公众号排版设计器变更记录

## 需求来源

用户要求在桌面创建一个新项目，调研市面主流公众号设计器功能，规划并落地一个可直接使用的公众号排版设计器。项目需要支持导出适合公众号的内容，预留 B 端广告收益位置，并面向百度搜索做 SEO。

## 市场调研摘要

- 135 编辑器强调公众号模板、素材库、AI 排版和新媒体内容生产能力。
- 秀米类工具以图文排版、原创模板素材和多样化风格为核心。
- 微排版强调 SVG 互动图文、拖拽画布、自定义动画和实时预览。
- 壹伴、美编类工具扩展到多账号运营、热点素材、AI 写作、图文诊断等运营流程。
- Markdown 公众号编辑器强调低学习成本、所见即所得、一键复制到微信编辑器。

## 架构理解

- 项目是从零创建的静态 Web 应用。
- 无构建依赖，适合直接部署到 GitHub Pages。
- 本地开发通过 `server.mjs` 提供静态资源。
- 所有文章草稿保存在浏览器 `localStorage`。
- 当前不接微信官方发布接口，不做账号体系和在线数据库。

## 实现内容

- 三栏工作台：左侧编辑，中间微信预览，右侧样式/广告/导出。
- 富文本编辑区：支持插入标题、正文、引用、提示、清单、图片占位和分割线。
- 三套样式主题：清爽专业、杂志专栏、产品发布。
- 三套文章模板：知识科普、产品介绍、活动通知。
- 公众号兼容导出：将编辑内容转换为带内联样式的 HTML。
- 媒体元素：支持本地图片、远程图片和视频卡片插入。
- 复制能力：复制富文本到微信后台，或复制 HTML 源码。
- B 端广告：支持配置广告主、文案、链接，并在导出中带赞助推荐卡片。
- SEO：标题、描述、关键词、结构化数据、canonical、robots、sitemap 和页面内容段落。

## 二次增强

- 在原有三套主题基础上扩展为 12 套首发主题。
- 新增主题选择器，支持通过下拉选择所有主题。
- 新增自定义主题色、浅底色和正文色配置。
- 将原本 3 个模板按钮升级为模板资源库。
- 模板资源库支持关键词搜索和场景筛选。
- 首发模板扩展为 12 个，覆盖知识科普、商业转化、运营活动、品牌内容和社群私域。

## 三次增强

- 新增媒体元素面板。
- 支持选择本地图片并插入到编辑器。
- 支持填写远程图片链接和图片说明。
- 支持填写视频标题、视频链接和封面图片链接，插入视频卡片。
- 导出时图片转换为公众号可粘贴的内联 `img`。
- 导出时视频转换为封面/链接卡片，避免外部 `iframe` 或 `video` 被微信后台过滤。

## 四次增强

- 新增六宫格插图块，适合产品卖点、步骤说明和案例拆解。
- 六宫格默认生成 6 张内置 SVG 插图和对应说明。
- 导出时六宫格转换为微信兼容的 2 列内联布局。
- 新增广告插入 300 字可见正文检测。
- 不使用隐藏透明文字凑字数，改为提供“补充可见说明”按钮。
- 导出和复制前如果广告开启且正文不足 300 字，会提示补充可见内容。

## 五次增强

- 统一“插入图片”和“编辑区图片”的工作流。
- 点击编辑区图片后，右侧媒体面板自动加载图片链接和说明。
- 修改右侧图片链接或说明后，可以直接更新所选图片。
- 点击视频卡片后，右侧面板自动加载视频标题、链接和封面。
- 点击六宫格中的单个卡片后，可以用图片面板替换该卡片图片和标题。
- 当前选中的媒体块在编辑区有高亮边框。

## 图片链接修复

- 修复带查询参数图片链接无法预览的问题。
- 原因是内部保存图片地址时提前把 `&` 转义为 `&amp;`，导致浏览器实际请求错误地址。
- 现在内部保存原始 URL，只在生成 HTML 属性时做转义。
- 已用百度图片链接验证：编辑区和预览区均能加载 `1280x960` 图片。

## UI 工作台改版

- 将页面视觉从偏展示型风格调整为更克制的 B 端编辑工具风格。
- 重建 `src/styles.css` 的设计 token，包括背景、面板、边框、文字、状态色和阴影层级。
- 三栏布局改为更稳定的编辑区、预览区、控制区比例，并限制整体最大宽度。
- 左右面板继续保持粘性滚动，预览区保留微信宽度和手机宽度切换。
- 右侧控制区在 `1260px` 以下切换为两列，在窄屏下切换为单列，减少拥挤和文字溢出。
- 模块由嵌套卡片改为分隔线分组，降低视觉噪声。
- 工具栏、主题卡片、模板卡片、表单、广告字数状态和媒体选中状态统一为更紧凑的操作控件。
- 编辑区增加低对比度辅助底纹，保留富文本内容的可读性。
- 预览卡片增加更清楚的纸张感，但不改变导出的公众号 HTML。
- 修复移动端顶部操作区、工具栏和编辑器被横向裁切的问题。
- 移动端使用两行顶部操作区：示例、保存并排，复制按钮独占一行。
- 避免在移动断点使用 `100vw` 造成滚动条宽度叠加，改为百分比宽度控制容器。

## UI 改版影响范围

- 只改动静态页面样式和测试断言。
- 不改变 `src/app.js` 的导出、复制、模板、媒体编辑和广告字数检测逻辑。
- 不新增第三方依赖。
- 不改变 GitHub Pages 静态部署方式。

## 已知风险

- 百度搜索第一无法保证，只能通过技术 SEO、内容建设和外链运营提升概率。
- 微信公众号后台对外部 HTML/CSS 有过滤规则，最终发布前仍需在微信后台检查。
- 当前没有图片上传、云端保存、多人协作、账号体系和广告投放后台。
- `sitemap.xml` 和 canonical 中的域名上线后需要替换为真实域名。
- 本次 UI 改版未引入图标库，因此按钮仍以文字为主；后续如果加入设计系统，可统一换成图标按钮和 tooltip。
