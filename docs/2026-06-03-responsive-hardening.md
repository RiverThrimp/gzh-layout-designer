# 笔记本宽度响应式修复记录

## 需求来源

用户反馈在笔记本电脑上稍微缩窄窗口后，部分功能区域会被遮挡或不容易看到，需要检查并修复响应式设计。

## 问题定位

- 原布局在 `1260px` 以下就把右侧功能面板折到编辑区和预览区下方。
- 常见笔记本浏览器窗口缩窄到 1100-1250px 时，页面仍有空间展示三栏，但右侧样式、媒体、广告、模板和导出功能已经掉到首屏下方。
- 用户会感知为部分功能被遮挡或需要额外滚动才能找到。

## 实现内容

- 将桌面三栏的最小列宽从 `330 / 390 / 300` 调整为 `300 / 360 / 270`。
- 将右侧功能面板折叠断点从 `1260px` 下调到 `1080px`。
- 在 1080px 以上的笔记本宽度继续保持编辑、预览、功能三栏同屏。
- 在 1080px 以下使用“编辑 + 功能面板”的两列布局，预览区下移到第二行，避免功能面板被预览区压到页面下方。
- 在 920px 以下继续切换为单列，并调整为编辑、功能面板、预览的顺序，避免功能面板被长预览内容压到页面深处。
- E2E 增加 1180px 和 1024px 断点检查。

## Before / After

Before：1180px 左右的窗口下，右侧功能面板被折到页面下方；1024px 的两列布局中，第一行是编辑和预览，样式、媒体、广告和导出功能仍然在预览区下方。

After：1180px 窗口仍保持三栏，右侧功能面板和编辑/预览同屏；1024px 窗口进入两列后，第一行是编辑和功能面板，预览区在第二行；900px 及以下单列时，功能面板也排在预览区之前，无横向溢出。

## 影响范围

- 只修改 `src/styles.css` 的响应式断点和列宽。
- 不改变 HTML 结构、编辑器状态、导出逻辑、构建流程或 GitHub Pages 发布方式。

## 测试记录

- Chrome DevTools 本地验证：
  - 1180px：三栏同屏，右侧功能面板位于首屏，无横向溢出。
  - 1024px：切换为两列，右侧功能面板在首屏右侧，预览区下移，无横向溢出。
  - 900px：切换为单列，功能面板排在预览区之前，无横向溢出。
  - 移动端宽度：保持单列，无横向溢出。
- `npm run lint`：静态检查通过。
- `npm run test:e2e`：真实浏览器烟测通过，覆盖 1180px、1024px、900px 和移动端断点。
- `npm run build`：构建通过。

## 已知风险

- 1080-1180px 下三栏会更紧凑，但比隐藏右侧功能面板更符合工具型工作台的使用预期。
- 1080px 以下预览区会下移，优先保证编辑和配置功能可操作。
- 如果用户浏览器缩放比例很高，仍可能提前进入紧凑布局。

## 回滚方式

- 恢复 `src/styles.css` 中 `.workspace` 的三栏最小宽度。
- 将中间断点从 `1080px` 恢复为 `1260px`。
- 移除 E2E 中新增的 1180px 和 1024px 断点断言。
