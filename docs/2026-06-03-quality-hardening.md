## 本次发现

- 后续优化将主题、模板、常量拆到 `src/data.js`，将通用工具函数拆到 `src/utils.js`。
- 新增了代码块、表格、撤销/重做、自动保存等能力。
- 页面可以运行，但新增能力的 E2E 覆盖不足。
- 撤销/重做历史栈只覆盖部分插入操作，普通正文输入、模板切换、清空和媒体更新存在历史缺口。
- 文档没有同步新增模块和新增能力。
- 项目根目录存在 `.DS_Store`、`docs/.DS_Store`、`screenshot-test.png` 等本地杂项文件。

## 实现内容

- 修复编辑器历史栈：
  - 普通正文输入通过防抖写入历史。
  - `Cmd/Ctrl+Z` 前先捕获当前编辑状态。
  - 模板切换、清空草稿、图片更新、六宫格卡片更新、视频更新都会在变更后写入历史。
- 升级 `tests/e2e-smoke.mjs`：
  - 继续检查静态 HTML、模块文件和 SEO 基础文件。
  - 使用本机 Chrome headless 与 DevTools Protocol 执行真实浏览器流程。
  - 验证页面渲染出 12 个主题和 12 个模板。
  - 点击代码块和表格按钮，验证编辑区和预览区都出现对应结构。
  - 切换移动端视口，验证没有横向溢出，且移动端复制按钮显示为短文案。
- 新增 `.gitignore`。
- 删除本地杂项文件：
  - `.DS_Store`
  - `docs/.DS_Store`
  - `screenshot-test.png`
- 更新组件文档，补充撤销/重做、自动保存、代码块和表格说明。

## 已知风险

- `index.html`、`robots.txt`、`sitemap.xml` 仍需要在正式上线前替换为真实域名。
- 真实浏览器 E2E 默认使用 macOS Chrome 路径；如果在其他环境运行，需要设置 `CHROME_PATH`。
- 目录目前还不是 Git 仓库，无法提供 Git diff 或提交记录。

## 回滚方式

- 回滚历史栈修复：恢复 `src/app.js`。
- 回滚 E2E 升级：恢复 `tests/e2e-smoke.mjs`。
- 回滚部署卫生：删除 `.gitignore`，但不建议恢复本地杂项文件。
