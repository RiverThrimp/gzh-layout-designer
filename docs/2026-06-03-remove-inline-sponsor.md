# 移除文章内赞助推荐记录

## 需求来源

用户澄清“赞助推荐”不是要添加在用户正在编辑和导出的公众号文章里，要求先移除该模块并重新发布到 GitHub。

## 实现内容

- 移除右侧“商业广告位”配置模块。
- 移除导出区域的广告字数检测和“补充可见说明”按钮。
- 移除导出 HTML 中自动追加赞助推荐卡片的逻辑。
- 移除复制、复制 HTML、下载 HTML 前的广告字数提醒。
- 移除草稿保存中的广告配置字段读取和写入。
- 调整内置模板中与文章内广告卡片相关的文案，改为行动入口或内容组件。
- 调整公开指南、更新日志和组件说明中的旧广告位描述。
- 更新测试断言，确保首页和公开内容页不再出现已移除的文章内赞助推荐提示。

## Before / After

Before：用户复制或导出文章时，如果广告开关开启，会把赞助推荐卡片一起带入公众号文章。

After：用户导出的公众号文章只包含用户编辑的正文、媒体、模板和样式，不再自动带产品自己的赞助推荐内容。

## 影响范围

- 修改 `index.html`、`src/app.js`、`src/data.js`、`src/styles.css`。
- 修改 `guides/copy-to-wechat.html`、`guides/wechat-layout-tips.html`、`changelog/index.html` 和 `docs/components/editor-workspace.md`。
- 修改静态检查和 E2E 断言。
- 不改变主题、模板库、媒体插入、复制富文本和构建发布流程。

## 测试记录

- `npm run lint`：通过，静态检查确认广告模块 DOM、导出补充按钮、字数检测样式和广告构建逻辑均已移除。
- `npm run test:e2e`：通过，本地源码服务烟测通过。
- `npm run build`：通过，生成 `dist/index.html`、`assets/app.0c9f21c252.js`、`assets/styles.cb0ffd96e1.css`。
- `npm run test:dist`：通过，本地发布产物烟测通过。
- `npm run test:live`：未通过。线上 GitHub Pages 当前仍是旧版 `gh-pages` 构建，仍包含“商业广告位”。需要提交并推送新构建产物后重新验证。

## 已知风险

- 如果用户浏览器里已有旧草稿的广告字段，本版本会忽略这些字段，不再恢复到 UI。
- 后续如需商业合作入口，应放在网站自己的合作页或反馈入口，而不是用户文章导出内容里。

## 回滚方式

- 恢复右侧商业广告模块。
- 恢复 `buildAdHtml()`、广告字数检测和导出拼接逻辑。
